## シリーズの背骨：AIセキュリティ3原則

**「分ける・防ぐ・残す」の順**でシリーズ全体の背骨として採用。各回で「今日はどの原則に対応するか」を冒頭で提示する。

| 順 | 原則 | 内容 | メインで扱う回 | 時系列上の位置 |
|---|------|------|----------------|----------------|
| 01 | **分ける** | エージェントの動作環境・アカウント・権限をメイン環境から分離する | 第2回（環境） | **事前**：予防 |
| 02 | **防ぐ** | 悪意あるものを入れない、意図しない操作を止める | 第3回（基本ガード） | **実行時**：制御 |
| 03 | **残す** | エージェントの行動ログを記録し、監査・評価できる状態にする | 第6回（ログ/トレース） | **事後**：監査 |

## 第1回〜第6回の3原則リンケージ

```
第1回「セキュリティ」：3原則全体を提示
    ├─ 入口・出口・境界の3リスクカテゴリ
    └─ 「次回以降で各原則の実装を学ぶ」と宣言
         │
         ├─ 第2回「環境」 → 3原則の【01 分ける】  ★事前：予防
         │    ├─ 4段階の分離（プロセス/コンテナ/VM/物理マシン）
         │    ├─ DevContainer と Claude Code Sandbox は別レイヤー
         │    └─ クラウドIAM最小権限
         │
         ├─ 第3回「基本ガード」 → 3原則の【02 防ぐ】  ★実行時：制御
         │    ├─ 助言的→決定的の階段（CLAUDE.md→Permission→Sandbox→Hooks）
         │    ├─ settings.json に4レイヤーを統合
         │    ├─ Permission Deny だけでは Bash を止められない（Sandboxとセット必須）
         │    └─ Bypass モード対策（failIfUnavailable: true）
         │
         ├─ 第4回「自動化・共有」 → 【防ぐ】の延長（自動化で強化）
         │    ├─ Skills（必要時のみ読込、トークン節約）
         │    ├─ MCP → Skills 変換（80-98% トークン削減）
         │    ├─ MCPセキュリティ（.mcp.jsonリポジトリ経由リスク、enableAllProjectMcpServers: false）
         │    ├─ スラッシュコマンド & Plugin（型のチーム配布）
         │    └─ コンテキストエンジニアリング（圧縮/構造化ノート/サブエージェント）
         │         └─ 「コンテキストウィンドウの中身は揮発する。ファイルは残る。」
         │
         ├─ 第5回「AIとの仕事の型」 → 【残す】への橋渡し
         │    └─ 計画→実装→記録のループ、SDD、AIDLC
         │
         └─ 第6回「ログ/トレース」 → 3原則の【03 残す】  ★事後：監査
              └─ Langfuseによる自動的なトレース・評価・改善
```
