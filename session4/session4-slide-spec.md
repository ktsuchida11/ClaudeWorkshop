## S4: Skills ― トークンを節約しながら知識を提供する（4-6分）★重要

上段：動作フロー
```
タスクを受け取る → description 照合 → SKILL.md 読み込み → 手順適用 & 実行
```

中段：CLAUDE.md vs Skills 比較表

| | CLAUDE.md | Skills |
|---|---|---|
| 読み込み | **毎回全文** | **必要時のみ** |

| 適した内容 | プロジェクト全体のルール | ファイルタイプ別・機能別の手順 |
| 分離の目安 | 100〜200行以内 | 長い手順は Skills に分離 |

下段：Skill ファイル構成
```

├── SKILL.md          ← frontmatter + Markdown
└── scripts/          ← 補助スクリプト
```
frontmatter: `name`（/スラッシュコマンド名にもなる）/ `description`（自然言語マッチでトリガー）

## S7: スラッシュコマンド & Plugin ― 型をチームで共有（10-12分）

左：スラッシュコマンド
- `.claude/commands/` にMarkdownで定義
- `/deploy`, `/review`, `/test-all` のように呼び出し
- プロンプト依存（ユーザーが明示的に実行）
- 変数展開: `$ARGUMENTS` で引数を受け取り
- 活用例: デプロイ手順 / レビューチェックリスト / テスト一括実行

右：Plugin
- Skills + Hooks + Commands + MCP を**まとめてパッケージ化**
- チームメンバーに一括配布可能
```
plugin-name/
├── .claude-plugin/plugin.json
├── commands/
├── agents/ | skills/ | hooks/
└── .mcp.json
```
- **第3回のsettings.jsonもPlugin化できる** → セキュリティ設定をチーム展開

下部:
> Skills = 知識の自動提供 / コマンド = ワークフローの定型化 / Plugin = 全部まとめて配布

## S8: コンテキストエンジニアリング ― 圧縮と外部化（12-13分）

タイトル: コンテキストエンジニアリング ― 圧縮と外部化
サブタイトル: コンテキストウィンドウには限りがある。「忘れても大丈夫」な仕組みを作る

左：戦略1 ― 圧縮（/compact と /clear）
- `/compact`: コンテキストを要約して量を減らす
  - ⚠ 必要な情報も消える → **先にアウトプットをファイルに残してから**
  - 作業の区切りで実行
- `/clear`: 全リセット。別の作業に完全切替時

右：戦略2 ― 構造化ノート（ファイルに残す）★今回の修正ポイント
- **作業内容・計画・決定事項をファイルに書き出す**
- `/clear` しても、ファイルは残る → **次のセッションで継続できる**
- **自分も忘れる** → 1週間後に「何やったっけ」を防げる
- 残すべきファイルの例：
  - `docs/plan.md` ― 作業計画・タスク分解
  - `docs/decisions.md` ― 設計判断とその理由
  - `docs/progress.md` ― 作業ログ・ブロッカー
- CLAUDE.md に「このファイルを参照して」と書けば、新セッションでも文脈を引き継げる

下部:
> **「コンテキストウィンドウの中身は揮発する。ファイルは残る。」**
> だから重要な情報はファイルに出す。これが第5回「バイブコーディング回避」のドキュメント文化にもつながる。

## S9: コンテキストエンジニアリング ― 戦略3: サブエージェント化（13-15分）

左：仕組みと利点
- サブエージェントは**独立したコンテキストウィンドウ**で動作
- 親エージェントのコンテキストに影響しない
- 結果はファイル経由で親にレポート
- ⚠ 実行単位でコンテキストを忘れる → ファイルで共有が必須

右：推奨パターン（用途別3カテゴリ）

| 用途 | サブエージェント例 |
|---|---|
| 成果物チェック | Code-Reviewer / Test-Analyzer / Security-Checker |
| 情報収集 | Impact-Analyzer / Dependency-Checker / File-Finder |
| 大量処理 | Test-Runner / Build-Executor / Log-Analyzer |
