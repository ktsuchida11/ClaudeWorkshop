# Claude Code 勉強会 全10回シリーズ

**対象**：Claude Code を日常的に使っている人と未経験者が混在するクラス
**形式**：30分 × 全10回（週1開催）
**講師**：土田 晃司（Koji Tsuchida）

---

## このシリーズの背骨：3原則「分ける・防ぐ・残す」

```
01  分ける  ─────  動作環境・アカウント・権限を、メイン環境から分離する
02  防ぐ    ─────  悪意あるものを入れない、意図しない操作を止める
03  残す    ─────  エージェントの行動ログを記録し、監査・評価できる状態にする
```

> 出典: GMOフラットセキュリティ CTO 米氏 対談より
> 順序は本シリーズの構成上の都合で入れ替え

## 各回の概要

### 第1回 セキュリティ（11枚 + 用語辞典）

> **「分ける・防ぐ・残す、だけ覚えて帰る」**

シリーズ全10回の背骨となる3原則を導入する回。Claude Code で起こりうる「入口・出口・境界」の3つの脅威カテゴリを整理し、Deny ルールと PreToolUse Hook の違い（助言的 vs 決定的）を実演で見せる。

- **学ぶこと**：3つの脅威カテゴリ／3原則の対応関係／Deny と Hooks の使い分け
- **実演**：Deny ルールでブロック → Hook で決定的に止める対比デモ
- **次回への接続**：第2回「環境（分ける）」の実践編へ

### 第2回 環境（13枚 + 用語辞典）

> **「迷ったら DevContainer から始める」**

3原則の「01 分ける」を物理・論理レベルで実装する回。プロセス → コンテナ → VM → 物理マシンの 4 段階の分離レベルを比較し、DevContainer と Claude Code Sandbox の関係（部屋の中でさらに手袋）を解説する。

- **学ぶこと**：4段階の分離レベル／DevContainer vs Sandbox／クラウドIAMの最小権限
- **実演**：ローカル直接 vs DevContainer ― 同じ依頼での挙動の違い
- **次回への接続**：第3回「基本ガード（防ぐ）」へ

### 第3回 基本ガード（11枚 + 用語辞典 ※再構成版）

> **「settings.json 1ファイルをコピーする」**

3原則の「02 防ぐ」を settings.json で実装する回。**指示の追従性4つの階段**（CLAUDE.md → Permission → Hooks → Sandbox）を見取り図として最初に提示し、CLAUDE.md / Permission / Sandbox / Hooks をそれぞれ詳説する構成。

- **学ぶこと**：4つの階段の強制力の違い／統合 settings.json の構成／Bypass モードの危険性
- **実演**：統合 settings.json + /sandbox を全部入れた状態で動かす
- **構造変更**：旧 S3「4つの防御レイヤー」を削除し、S8「4つの階段」を S4 に昇格して見取り図とする

### 第4回 自動化・共有（12枚 + 用語辞典）

> **「繰り返し作業は Skill にする」**

防ぐの延長で、自分の型を資産化する回。**主軸は2つ：Skills と コンテキスト設計**。MCP / コマンド / Plugin は「Skills の仲間」として位置づける。MCP→Skills 変換でトークン 80-98% 削減できることを示す。

- **学ぶこと**：Skills の仕組み／MCP の落とし穴／コンテキストエンジニアリング3戦略
- **実演**：Skill Creator で Skill を作って動かす（5分で1つ）
- **次回への接続**：第5回「AIとの仕事の型」へ

### 第5回 AIとの仕事の型（12枚 + 用語辞典）

> **「計画を書いてから AI に渡す」**

残すへの橋渡し。バイブコーディングの罠と、それを防ぐ5つのアクション（要件／ドキュメント／計画／テスト／レビュー）を学ぶ。バイブ → SDD → AIDLC の成熟度モデルで自分の現在地を知る。

- **学ぶこと**：4つの罠／5つの防止アクション／3段階成熟度モデル
- **実演**：要件 → 計画 → 実装 → レビュー → 記録の一連フロー
- **次回への接続**：第6回「ログ／トレース」へ

### 第6回 ログ／トレース（14枚 + 用語辞典 ※スライド分割版）

> **「計器なしで本番に飛ばさない」**

3原則の「03 残す」をシステムで実装する回。**「困った経験」スライド**で共感を作ってから LLM Observability の説明へ。Langfuse の3機能（Trace / Evaluate / Prompt CMS）を学ぶ。**「Claude Code のトレースを取る」スライドは2枚に分割**し、課題（3つに厳選）とトレース例を別スライドにした。

- **学ぶこと**：従来監視 vs 生成AI監視／Langfuseの3機能／Trace→Evaluate→Improve サイクル
- **実演**：Claude Code × Langfuse トレース画面でのデバッグ
- **構造変更**：S5「困った経験」を新規挿入／S9-S10 を分割

### 第7回 ユースケース（13枚 + 用語辞典）

> **「コードではなく『型』を実装する」**

後半「活用を広げる」フェーズの第1回。3つのレイヤー（対話 / Skills&コマンド / Routines&SubAgents）を導入し、5つのユースケース（コードベース探索 / データ変換 / Skill Creator / Skills 3つの主要型 / SubAgents&Routines）を解説。**Skill の8つの型は3つの主要型（Reference / Task / Document）に絞り**、残り5型はGitHubリファレンスへ退避。

- **学ぶこと**：3つのレイヤー／5つのユースケース／開発ワークフロー全体像
- **実演**：コードベース探索 → バグ発見 → Issue 登録（GitLab）
- **構造変更**：8つの型 → 3つの主要型に削減（情報密度の改善）

## 各回スライドの構成パターン

各回30分の構成は次のテンプレートに従っています：

```
[スライド10-15分]
  S1.  表紙（核フレーズ提示）
  S2.  用語ミニ辞典（その回で初出するカタカナ語5つを1行で）
  S3.  前回までの地図 + 今日の位置
  S4.  今日の主題スライド ★重要
  S5-S(n-3).  個別の解説スライド
  S(n-2).  実演導入
[実演5-10分]
  S(n-1).  Key Takeaway（核フレーズ + 持ち帰るチェック項目 + 3原則の現在地）
[質疑5分 / 振り返り5分]
  S(n).  次回予告
```

### 共通の補助要素

- **核フレーズ**：表紙とKey Takeawayの2箇所で同じ15文字以内のフレーズを反復
- **用語ミニ辞典**：各回の初出カタカナ語を5つに絞った参照スライド
- **3原則の現在地**：Key Takeawayに進捗マーク（✅完了 / 🎯今日 / ⏳次回以降）
- **💡 補足**：本編で深追いしないものは「→ 詳しくは FAQ Qx-y」に置換

## 「難解」FB への対応方針（要約）

参加者からの「難解」というフィードバックの正体を分析した結果、**内容そのものではなく「30分1本というメディアと、スライドの情報密度・用語密度のミスマッチ」**が原因と特定。次の5本柱で対応：

1. **1スライド = 1メッセージに削る**（情報を半分に絞り、詳細は話者ノート / FAQ へ退避）
2. **カタカナ語の初出ルール**（用語ミニ辞典スライドを各回頭に配置）
3. **背骨スライドを本編中に2回以上戻す**（3原則の現在地マークで現在地を明示）
4. **30分の中で1つだけ覚える1文を全回で固定**（核フレーズの3箇所反復）
5. **公開版とプレゼン版を分ける**（GitHub公開資料は読み物として完成度高く、本編は思い切って削る ― 中長期）

詳細は `claude-code-workshop-review.md` 参照。
