# Claude Code 勉強会シリーズ 全体構成 v2

**対象**：Claude Codeを日常的に使っている人と未経験者が混在するクラス
**形式**：30分 × 全10回（週1開催）
**30分の内訳**：スライド10-15分 / 実演5-10分 / 質疑5分 / 振り返り5分

---

## シリーズの背骨：AIセキュリティ3原則

GMOフラットセキュリティ 米氏が提示する**「分ける・残す・防ぐ」**をシリーズ全体の背骨として採用する。各回で「今日はどの原則に対応するか」を冒頭で提示する。

| 原則 | 内容 | メインで扱う回 |
|------|------|----------------|
| **分ける** | エージェントの動作環境・アカウント・権限をメイン環境から分離する | 第2回（環境） |
| **残す** | エージェントの行動ログを記録し、監査・評価できる状態にする | 第6回（ログ/トレース） |
| **防ぐ** | 悪意あるものを入れない、意図しない操作を止める | 第3回（基本ガード） |

第1回は3原則の全体像を示し、以降の各回で「この原則の実装編」として位置づける構成。

---

## シリーズ全体の3つの地図

参加者が毎回迷子にならないよう、シリーズ全体を貫く3つのフレームを初回で提示し、以降の各回で「今日はこの座標のここ」と示す。

### 地図1：AIセキュリティ3原則（分ける・残す・防ぐ）
→ 前半（1〜6回）の背骨

### 地図2：3つの課題フレームワーク（既存スライドA-S37より）
- メモリ問題（過去を忘れる）
- 推論問題（意図通り動かない）
- 外部連携問題（ツール選択の精度低下）
→ 機能編（3〜4回）の整理軸

### 地図3：AI開発の3段階成熟度モデル（既存スライドA-S45より）
- Level 1: バイブコーディング
- Level 2: スペック駆動開発（SDD）
- Level 3: AIDLC
→ 後半（5〜7回）の到達点

第1回「セキュリティ」：3原則全体を提示
    ├─ 入口・出口・境界の3リスクカテゴリ

         │

         │
         ├─ 第3回「基本ガード」 → 3原則の【防ぐ】

         │

## 第2回 環境 ― 30分構成詳細

### Key Takeaway
「分離には4段階ある（プロセス/コンテナ/VM/物理マシン）。DevContainerが基本、ローカルはHooks+Permissionで代替、VS Code拡張は権限境界を理解した上で使う」

### スライド構成（10-15分）

| 時間 | スライド | 内容 |
|------|---------|------|
| 0-2分 | S1: 導入 | 前回の3原則のうち「分ける」を深掘りする回 |
| 2-5分 | S2: 分離の4段階 | プロセス分離 / コンテナ分離 / VM分離 / 物理マシン分離 |
| 5-10分 | S3-4: 環境別マトリクス | ローカル直接 / VS Code拡張 / DevContainer / 別マシン（Mac mini等）の権限境界比較 |
| 10-13分 | S5: IAMとクラウド | エージェント専用アカウント・最小権限の原則・想定外IAMロールの罠 |
| 13-15分 | S6: 選択ガイド | どの場面でどの環境を選ぶか |

### 環境別マトリクス（S3-4の内容）

| 環境 | ファイルアクセス | ネットワーク | 認証情報到達 | 推奨度 | 使うべき場面 |
|------|-----------------|-------------|-------------|--------|-------------|
| ローカル直接 | 全て | 全て | 高（ブラウザCookie等） | △ | 検証・個人ツール |
| VS Code拡張 | ワークスペース中心 | 全て | 中 | △ | 軽い編集 |
| DevContainer | コンテナ内のみ | 制御可 | 低 | ◎ | 業務コード |
| 別マシン（Mac mini等） | 物理隔離 | 物理隔離 | 極低 | ◎◎ | 高リスクタスク |

### 実演（8分目安）

- DevContainer起動 → 同じタスクをローカルと比較
- DevContainer内からはホストの`.ssh`にアクセスできないことを見せる
