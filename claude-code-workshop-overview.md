## 各回の30分構成（v9 確定版・第8回以降も踏襲）

### 共通構造（全回テンプレート）

```
[スライド10-15分]
  S1.  表紙（核フレーズ提示）
  S2.  用語ミニ辞典 ← v9で追加
  S3.  前回までの地図 + 今日の位置（v10：第8・9回は「今日の位置と本日の流れ」1枚に統合）
  S4.  今日の主題スライド ★重要
  S5-S(n-3).  個別の解説スライド
  S(n-2).  実演導入
[実演5-10分]
  S(n-1).  Key Takeaway（核フレーズ + チェック項目 + 3原則の現在地 ← v9で追加）
[質疑5分 / 振り返り5分]
  S(n).  次回予告
```

### 第8回 自己改善エージェント（30分版21枚 / 詳細版28枚）

題材は「市場レポート（為替など）の日次配信」を AI が下書きし、筆者の実レポート（＝正解）と毎日突き合わせて書き方ルールを自動更新する自己改善エージェント。核フレーズ「正解があれば毎日賢くなる」。

- **学ぶこと**：正解（教師データ）の定義／生成と評価の分離／style_guide の学習機構／破綻を防ぐ4原則／成果（スコア7→13）
- **30分版の構成**：実演なし前提でスライドを厚めに保持。実装の深掘り（6層構造・Step詳細・情報収集M/N/I/B・情報検索・ファクトチェック2層・Weekly Reviewer・スキル粒度）の7枚を詳細版へ退避
- **次回への接続**：第9回「開発へのループ応用」へ

### 第9回 開発へのループ応用（30分版18枚 / 詳細版22枚）

第8回で確立したハーネスの「型」を開発工程のループへ移し替える。核フレーズ「実装と検証をISSUEで繋ぐ」。

- **学ぶこと**：ループ設計の4原則（評価基準・止め方・役割分担・判断）／二重ループの移し替え／PGE（Planner-Generator-Evaluator）／実装役と検証役の分離と ISSUE 連携／検証ゲートと /goal
- **強調点**：実装役（Generator）と検証役（Evaluator）を物理的に分け、受け入れ条件＝gold_standard も検証結果も **ISSUE テキストに残して往復**する（非同期・記録・責務分離）
- **30分版の構成**：概念の移し替え（重複）・向く/向かないタスク・私の現在地・おまけCodex比較の4枚を詳細版へ退避
- **次回への接続**：第10回「Agent Teams」へ

### 第7回 ユースケース（13枚 ※実pptx化）

S7「Skillの8つの型」を「3つの主要型（Reference / Task / Document）+ 残り5型は参考」に削減：

0-2分 表紙 + 用語辞典 → 2-4分 前回振り返り → 4-6分 3つのレイヤー★ → 6-8分 UC①コードベース探索 → 8-10分 UC②データ変換 → 10-12分 UC③ Skill Creator → 12-13分 UC④ Skills 3つの主要型 → 13-15分 UC⑤ SubAgents & Routines → 15-17分 全体フロー図★核心 → 17-23分 実演 → 23-28分 質疑 → 28-30分 振り返り
