## 各回の30分構成（v9 確定版）

### 共通構造（全回テンプレート）

```
[スライド10-15分]
  S1.  表紙（核フレーズ提示）
  S2.  用語ミニ辞典 ← v9で追加
  S3.  前回までの地図 + 今日の位置
  S4.  今日の主題スライド ★重要
  S5-S(n-3).  個別の解説スライド
  S(n-2).  実演導入
[実演5-10分]
  S(n-1).  Key Takeaway（核フレーズ + チェック項目 + 3原則の現在地 ← v9で追加）
[質疑5分 / 振り返り5分]
  S(n).  次回予告
```

### 第3回 ユースケース（12枚 ※構造変更）

旧 S3「4つの防御レイヤー」を削除、旧 S8「指示の追従性 4つの階段」をS4に昇格して見取り図化：

0-2分 表紙 + 用語辞典 → 2-4分 前回振り返り → 4-6分 **指示の追従性4つの階段★（旧S8昇格）** → 6-12分 CLAUDE.md / Permission / Sandbox / Hooks の各詳説 → 12-15分 統合settings.json → 15-23分 実演 → 23-28分 質疑 → 28-30分 振り返り

### 第4回 自動化・共有（13枚）

S4を「主軸2つ ― Skills と コンテキスト設計」に再フレーミング。MCP/コマンド/Pluginは「Skillsの仲間」として位置づけ。

### 第6回 ログ/トレース（15枚 ※スライド分割）

S5に「あれ、なんで？」共感スライドを挿入。S9-S10 を「Claude Code 特有の課題」と「トレース例」の2枚に分割：

0-5分 表紙 + 用語辞典 + 前回振り返り + 従来 vs 生成AI監視 → **5-7分 困った経験（v9新規）** → 7-15分 LLM Observability → Langfuse → 機能①トレース → **15-19分 Claude Code特有の課題★ + トレース例★（v9で2分割）** → 19-23分 機能②評価 + 機能③Prompt CMS → 23-28分 実演 → 28-30分 振り返り

### 第7回 ユースケース（13枚 ※実pptx化）

S7「Skillの8つの型」を「3つの主要型（Reference / Task / Document）+ 残り5型は参考」に削減：

0-2分 表紙 + 用語辞典 → 2-4分 前回振り返り → 4-6分 3つのレイヤー★ → 6-8分 UC①コードベース探索 → 8-10分 UC②データ変換 → 10-12分 UC③ Skill Creator → 12-13分 UC④ Skills 3つの主要型 → 13-15分 UC⑤ SubAgents & Routines → 15-17分 全体フロー図★核心 → 17-23分 実演（コードベース探索→バグ発見→Issue登録） → 23-28分 質疑 → 28-30分 振り返り

## v9で確定した設計判断

### スタイル仕様（v9で全7回統一）

| 用途 | 背景色 | 見出し色 | 本文色 | 強調 |
|---|---|---|---|---|
| カバー / DEMO / Key Takeaway | `#1E2761` | `#FFFFFF` | `#CADCFC` | `#FFC800` |
| コンテンツ / 用語ミニ辞典 / Next | `#FFFFFF` | `#1E2761` | `#2D3748` | `#FFC800` |
| 装飾的な大型数字 | – | `#151B47` | – | – |

### 核フレーズ運用ルール

- **15文字以内**で、表紙「★ 覚えて帰る一言: 「XX」」と Key Takeaway の冒頭で同一文言を反復
- ゴールド `#FFC800` で強調

### 用語ミニ辞典運用ルール

- 各回 **slide 2** に固定配置（前回振り返りの前）
- **カタカナ語5個まで**、各1行の定義
- 白背景 + ネイビー見出し + ダーク本文（コンテンツ系スタイル）

### 3原則の現在地マーク運用ルール

- **Key Takeaway スライドの最下部** に固定配置
- フォーマット：`01 分ける [STATE]    02 防ぐ [STATE]    03 残す [STATE]`
- STATE：`✅` 完了、`🎯` 今日、`⏳` 次回以降、`→` 橋渡し中、`✅+` 延長中

### 💡注釈の運用方針

- **コアメッセージ**：本編に残す（受講者がスライドだけ見ても核が伝わる）
- **補足・橋渡し・詳細**：「→ 詳しくは FAQ Qx-y」形式に置換（深追いは FAQ で）
