# Claude Code 勉強会シリーズ 全体構成 v4

**対象**：Claude Codeを日常的に使っている人と未経験者が混在するクラス
**形式**：30分 × 全10回（週1開催）
**30分の内訳**：スライド10-15分 / 実演5-10分 / 質疑5分 / 振り返り5分

---

## v4の変更点（v3からの差分）

- **第3回のスライド順序を変更**：CLAUDE.md → Permission → Sandbox → Hooks の順に（助言的→決定的の階段を登る構成）
- **第3回の防御レイヤーを4要素に拡張**：Permission / Sandbox / Hooks / ネットワーク allowlist
- **settings.jsonを軸にした構成**に変更：個別機能を説明 → 最後に統合settings.jsonを見せる流れ
- **Sandboxの重要注意点を追加**：ファイルシステム制御はSandbox有効時のみ動作 / Bypassモードの危険性とfailIfUnavailableによる対策
- **Dagger Container Useは第8回「Agent Teams」で扱う**ことを明記

---

## シリーズの背骨：AIセキュリティ3原則

**「分ける・防ぐ・残す」の順**でシリーズ全体の背骨として採用。各回で「今日はどの原則に対応するか」を冒頭で提示する。

| 順 | 原則 | 内容 | メインで扱う回 | 時系列上の位置 |
|---|------|------|----------------|----------------|
| 01 | **分ける** | エージェントの動作環境・アカウント・権限をメイン環境から分離する | 第2回（環境） | **事前**：予防 |
| 02 | **防ぐ** | 悪意あるものを入れない、意図しない操作を止める | 第3回（基本ガード） | **実行時**：制御 |
| 03 | **残す** | エージェントの行動ログを記録し、監査・評価できる状態にする | 第6回（ログ/トレース） | **事後**：監査 |

## シリーズ全体の3つの地図

### 地図1：AIセキュリティ3原則（分ける・防ぐ・残す）
→ 前半（1〜6回）の背骨

### 地図2：3つの課題フレームワーク（既存スライドA-S37より）
- メモリ問題（過去を忘れる）
- 推論問題（意図通り動かない）
- 外部連携問題（ツール選択の精度低下）
→ 機能編（3〜4回）の整理軸

### 地図3：AI開発の3段階成熟度モデル（既存スライドA-S45より）
- Level 1: バイブコーディング
- Level 2: スペック駆動開発（SDD）
- Level 3: AIDLC
→ 後半（5〜7回）の到達点

## 第1回〜第6回の3原則リンケージ（背骨の見える化）

```
第1回「セキュリティ」：3原則全体を提示
    ├─ 入口・出口・境界の3リスクカテゴリ
    └─ 「次回以降で各原則の実装を学ぶ」と宣言
         │
         ├─ 第2回「環境」 → 3原則の【01 分ける】  ★事前：予防
         │    ├─ 4段階の分離（プロセス/コンテナ/VM/物理マシン）
         │    ├─ DevContainer と Claude Code Sandbox は別レイヤー
         │    └─ クラウドIAM最小権限
         │
         ├─ 第3回「基本ガード」 → 3原則の【02 防ぐ】  ★実行時：制御
         │    ├─ 助言的→決定的の階段（CLAUDE.md→Permission→Sandbox→Hooks）
         │    ├─ settings.json に4レイヤーを統合
         │    ├─ Permission Deny だけでは Bash を止められない（Sandboxとセット必須）
         │    └─ Bypass モード対策（failIfUnavailable: true）
         │
         ├─ 第4回「機能2」 → 【防ぐ】の延長（自動化で強化）
         │
         ├─ 第5回「バイブ回避」 → 【残す】への橋渡し
         │    └─ ドキュメント・計画・レビューで記録を残す
         │
         └─ 第6回「ログ/トレース」 → 3原則の【03 残す】  ★事後：監査
              └─ Langfuseによる自動的なトレース・評価・改善
```

後半（第7〜10回）は、3原則を前提とした上で**活用を広げる**フェーズ。
